🏥 案例一：智慧医院管理系统

用 OPERA 框架为一家三甲医院建立完整的 Ontology 模型

📋 业务背景

某三甲医院正在进行数字化转型，面临以下挑战：

  • HIS（医院信息系统）、LIS（化验系统）、PACS（影像系统）数据孤岛
  • 跨科室会诊时，医生无法快速了解患者全貌
  • 药物过敏事故偶有发生，缺乏自动预警
  • 新上任的 AI 团队想构建智能问诊助手，但缺乏结构化医疗知识

⚡ 第三步：E — 梳理事件/动作

核心业务动作
动作定义 
Action 挂号(Register) {
  触发者: Patient | 工作人员
  目标:   Patient → Department
  前置:   Doctor.available == true
  效果:   创建 Appointment, 分配 Doctor
}

Action 开处方(Prescribe) {
  触发者: Doctor
  目标:   Patient, Medication[]
  前置:   Doctor.treats(Patient)
  ⚠️ 安全检查: Medication ∉ Patient.allergies  // 关键！
  效果:   创建 Prescription, 扣减库存
}

Action 入院(Admit) {
  触发者: Doctor
  目标:   Patient → Ward
  前置:   Ward.available_beds > 0
  效果:   Patient.type = InPatient, Ward.occupied += 1
}

Event 药物过敏警报(AllergyAlert) {
  触发条件: 处方中药品 ∈ 患者过敏列表
  效果:     阻止处方提交, 通知医生
}

🔗 第四步：R — 建立关系

关系模型 
// 核心关系
Doctor  ──treats──→  Patient     // 医生治疗患者 (M:N)
Doctor  ──belongs_to──→  Department  // 医生属于科室 (N:1)
Patient ──admitted_to──→  Ward       // 住院于病房 (1:1)
Doctor  ──prescribes──→  Prescription // 开处方 (1:N)
Prescription ──contains──→  Medication // 包含药品 (1:N)
Prescription ──for_patient──→  Patient  // 处方属于患者 (N:1)
Patient ──has_diagnosis──→  Diagnosis  // 诊断 (M:N)

// 逆关系
Patient ──treated_by──→  Doctor     // treats 的逆
Ward    ──houses──→  Patient         // admitted_to 的逆

📏 第五步：A — 添加公理/约束

公理定义 
// 约束型公理
A1: Patient.patient_id is UNIQUE
A2: InPatient DisjointWith OutPatient   // 同一时间不能既住院又是门诊
A3: Prescription for_patient exactly 1 Patient
A4: Prescription contains min 1 Medication

// 推理型公理（药物安全）
A5: IF Prescription.contains(med) AND 
       Prescription.for_patient.allergies.includes(med.category)
     THEN RAISE AllergyAlert
     
// 推理型公理（自动分类）
A6: IF Doctor.treats ONLY (Patient AND age < 14)
     THEN Doctor is-a Pediatrician

Python (owlready2) 
from owlready2 import *

# 创建本体
onto = get_ontology("http://example.org/smart-hospital.owl")

with onto:
    # ===== O: 对象类型 =====
    class Person(Thing): pass
    class Patient(Person): pass
    class InPatient(Patient): pass
    class OutPatient(Patient): pass
    class Doctor(Person): pass
    class Nurse(Person): pass
    class Department(Thing): pass
    class Ward(Thing): pass
    class Medication(Thing): pass
    class Prescription(Thing): pass
    class Diagnosis(Thing): pass

    # ===== P: 属性 =====
    class has_name(DataProperty, FunctionalProperty):
        domain = [Person]; range = [str]
    class has_age(DataProperty, FunctionalProperty):
        domain = [Person]; range = [int]
    class has_allergy(DataProperty):
        domain = [Patient]; range = [str]
    class has_stock(DataProperty, FunctionalProperty):
        domain = [Medication]; range = [int]
    class med_name(DataProperty, FunctionalProperty):
        domain = [Medication]; range = [str]

    # ===== R: 关系 =====
    class treats(ObjectProperty):
        domain = [Doctor]; range = [Patient]
    class treated_by(ObjectProperty):
        inverse_property = treats
    class belongs_to(ObjectProperty, FunctionalProperty):
        domain = [Doctor]; range = [Department]
    class contains_med(ObjectProperty):
        domain = [Prescription]; range = [Medication]
    class for_patient(ObjectProperty, FunctionalProperty):
        domain = [Prescription]; range = [Patient]

    # ===== A: 公理 =====
    AllDisjoint([InPatient, OutPatient])

# ===== 创建实例并测试推理 =====
with onto:
    # 创建科室
    cardiology = Department("cardiology")

    # 创建医生
    dr_wang = Doctor("dr_wang")
    dr_wang.has_name = ["王医生"]
    dr_wang.belongs_to = [cardiology]

    # 创建患者
    patient_zhang = InPatient("patient_zhang")
    patient_zhang.has_name = ["张三"]
    patient_zhang.has_age = [45]
    patient_zhang.has_allergy = ["青霉素"]

    # 建立关系
    dr_wang.treats.append(patient_zhang)

    # 创建药品
    aspirin = Medication("aspirin")
    aspirin.med_name = ["阿司匹林"]
    aspirin.has_stock = [500]

    # 创建处方
    rx001 = Prescription("rx001")
    rx001.for_patient = [patient_zhang]
    rx001.contains_med = [aspirin]

# E: 药物安全检查（动作逻辑）
def check_allergy(prescription):
    patient = prescription.for_patient[0]
    allergies = patient.has_allergy
    for med in prescription.contains_med:
        if med.med_name[0] in allergies:
            return f"⚠️ 过敏警报：{patient.has_name[0]}对{med.med_name[0]}过敏！"
    return "✅ 安全检查通过"

print(check_allergy(rx001))
# 输出: ✅ 安全检查通过

# 查询: 谁在治疗张三？
print("治疗张三的医生:", patient_zhang.treated_by)
# 输出: 治疗张三的医生: [smart-hospital.dr_wang]

🛒 案例二：电商供应链管理

对供应商→仓库→商品→订单→物流全链路建模

📋 业务背景

某大型电商平台拥有 10000+ 供应商、500+ 仓库、数百万 SKU，面临：

  • 不同供应商提供的同款商品，属性描述不一致（如"颜色:红色" vs "color:红"）
  • 库存在途、在仓、已预定等多状态管理复杂
  • 缺货预警不及时，经常出现超卖
  • 物流调度缺乏全局视角

🗺️ OPERA 五步建模

🏷️ O — 对象类型
🏢 供应商 Supplier 🏭 仓库 Warehouse 📦 商品 Product 📑 SKU 🛍️ 订单 Order 📋 订单项 OrderItem 🚚 物流单 Shipment 📍 库存记录 Inventory 👤 用户 User 📂 商品类目 Category
📝 P — 核心属性
属性定义 
Product:   product_id, name, brand, category, base_price
SKU:       sku_id, color, size, weight, selling_price
Inventory: quantity_available, quantity_reserved, quantity_in_transit
Order:     order_id, status, total_amount, created_at, payment_method
Warehouse: wh_id, location, capacity, current_utilization
Shipment:  tracking_no, carrier, estimated_delivery, actual_delivery
⚡ E — 事件/动作
核心业务动作 
Action 下单(PlaceOrder):
  触发: User, SKU[], quantities[]
  前置: ALL SKU.inventory.available >= quantity
  效果: 创建 Order, 扣减 available, 增加 reserved

Action 发货(Ship):
  触发: 仓库系统
  前置: Order.status == "已支付"
  效果: 创建 Shipment, reserved -= qty, in_transit += qty

Event 库存预警(LowStockAlert):
  触发: Inventory.available < threshold
  效果: 通知采购, 自动生成采购建议
🔗 R — 关系
关系模型 
Supplier ──supplies──→ Product      // M:N
Product  ──has_sku──→ SKU           // 1:N
Product  ──in_category──→ Category   // N:1
Category ──parent_of──→ Category    // 层级关系(传递)
Warehouse──stores──→ Inventory     // 1:N
Inventory──of_sku──→ SKU           // N:1
Order    ──placed_by──→ User        // N:1
Order    ──has_items──→ OrderItem   // 1:N
OrderItem──of_sku──→ SKU           // N:1
Order    ──shipped_via──→ Shipment  // 1:N
Shipment ──from_warehouse──→ Warehouse // N:1
📏 A — 公理
业务规则 
A1: Order has_items min 1 OrderItem  // 订单至少一个商品
A2: SKU.selling_price > 0
A3: Inventory.available >= 0          // 不能负库存
A4: IF Order.total != SUM(OrderItem.price * qty)
     THEN RAISE DataInconsistencyError      // 金额校验
A5: Shipment.actual_delivery >= Order.created_at  // 时间逻辑

💻 Python 实现要点

Python — 库存预警推理 
class SupplyChainOntology:
    def __init__(self):
        self.products = {}
        self.inventory = {}
        self.orders = []
        self.alert_threshold = 50

    def place_order(self, user, items):
        """下单动作 — 含前置条件检查"""
        for sku_id, qty in items:
            inv = self.inventory[sku_id]
            if inv['available'] < qty:
                raise ValueError(f"超卖! {sku_id} 可用:{inv['available']}, 需要:{qty}")

        # 扣减库存
        for sku_id, qty in items:
            self.inventory[sku_id]['available'] -= qty
            self.inventory[sku_id]['reserved'] += qty
            self._check_low_stock(sku_id)

        order = {'user': user, 'items': items, 'status': 'pending'}
        self.orders.append(order)
        return order

    def _check_low_stock(self, sku_id):
        """库存预警事件 — 自动推理"""
        inv = self.inventory[sku_id]
        if inv['available'] < self.alert_threshold:
            print(f"🚨 库存预警: {sku_id} 仅剩 {inv['available']} 件!")
            print(f"   → 建议采购 {self.alert_threshold * 3} 件")

# 使用示例
sc = SupplyChainOntology()
sc.inventory["SKU-001"] = {'available': 80, 'reserved': 0, 'in_transit': 0}
sc.place_order("user_001", [("SKU-001", 35)])
# 输出: 🚨 库存预警: SKU-001 仅剩 45 件!
#       → 建议采购 150 件

📚 案例四：企业知识管理系统

构建企业知识图谱，实现专家发现和知识复用

📋 业务场景

一家拥有 5000+ 员工的科技公司，期望：

  • 快速找到某技术领域的内部专家
  • 了解哪些项目使用了特定技术
  • 新员工快速了解相关知识文档
  • 避免重复造轮子——发现已有的解决方案

🗺️ OPERA 模型

完整 OPERA 建模 
═══════ O: 对象类型 ═══════
Employee（员工）           Skill（技能）
Department（部门）         Project（项目）
Document（文档）           Technology（技术栈）
  ├── DesignDoc            KnowledgeArea（知识领域）
  ├── APIDoc
  └── Tutorial

═══════ P: 属性 ═══════
Employee: name, role, join_date, expertise_level
Skill:    skill_name, category, difficulty
Project:  project_name, status, start_date, end_date
Document: title, author, created_at, tags[], views

═══════ E: 事件/动作 ═══════
Action 查找专家(FindExpert):
  输入: Skill 或 Technology
  逻辑: 查找 Employee WHERE has_skill(target) AND expertise_level >= 3
  输出: 排序后的专家列表

Action 推荐文档(RecommendDocs):
  输入: Employee（新员工）
  逻辑: 找到员工所在项目 → 项目使用的技术 → 相关文档
  输出: 推荐阅读列表

═══════ R: 关系 ═══════
Employee ──has_skill──→ Skill         // M:N + 熟练度
Employee ──works_on──→ Project        // M:N
Employee ──authored──→ Document       // 1:N
Employee ──in_dept──→ Department      // N:1
Project  ──uses_tech──→ Technology    // M:N
Document ──covers──→ Technology      // M:N
Document ──related_to──→ Document     // M:N
Skill    ──part_of──→ KnowledgeArea  // N:1

═══════ A: 公理 ═══════
A1: Employee.expertise_level ∈ [1,5]
A2: Document must have authored_by min 1 Employee
A3: IF Employee works_on Project P
     AND P uses_tech T
     THEN Employee is candidate_expert_of T  // 推理专家
A4: IF Employee has_skill S AND expertise >= 4
     THEN Employee is Expert of S

💻 Python 实现：专家推荐引擎

Python — 知识图谱查询 
class KnowledgeOntology:
    def __init__(self):
        self.employees = {}
        self.projects = {}
        self.skills = {}
        self.documents = {}

    def find_expert(self, tech_name, min_level=3):
        """查找专家 — 基于 Ontology 关系推理"""
        experts = []
        for emp in self.employees.values():
            # 直接技能匹配
            for skill, level in emp['skills']:
                if tech_name.lower() in skill.lower() and level >= min_level:
                    experts.append({
                        'name': emp['name'],
                        'level': level,
                        'source': '直接技能'
                    })
            # 项目经验推理 (公理 A3)
            for proj_id in emp.get('projects', []):
                proj = self.projects.get(proj_id, {})
                if tech_name in proj.get('technologies', []):
                    experts.append({
                        'name': emp['name'],
                        'level': '推理(项目经验)',
                        'source': f"参与项目: {proj['name']}"
                    })
        return sorted(experts, key=lambda x: x.get('level',0), reverse=True)

    def recommend_docs(self, employee_id):
        """推荐文档 — 基于 Ontology 关系链"""
        emp = self.employees[employee_id]
        techs = set()
        # 通过项目找到相关技术
        for proj_id in emp.get('projects', []):
            techs.update(self.projects[proj_id].get('technologies', []))
        # 通过技术找到相关文档
        recommended = []
        for doc in self.documents.values():
            if techs & set(doc.get('covers', [])):
                recommended.append(doc)
        return sorted(recommended, key=lambda d: d['views'], reverse=True)

# 使用示例
kg = KnowledgeOntology()
kg.employees["e001"] = {
    'name': '李工', 'skills': [('Python', 5), ('Ontology', 4)],
    'projects': ['p001']
}
kg.projects["p001"] = {'name': '知识图谱平台', 'technologies': ['Python', 'OWL', 'Neo4j']}

experts = kg.find_expert("Python")
print("Python 专家:", experts)
# [{'name': '李工', 'level': 5, 'source': '直接技能'},
#  {'name': '李工', 'level': '推理(项目经验)', 'source': '参与项目: 知识图谱平台'}]
